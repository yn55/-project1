Summary

This project has GitHub actions integrated into the code. There are two github work flows that are set up in the project:
dev.yml
prod.yml
When writing code on a branch, a user can commit and push their code into a github repository. The Github repository will receive a pull request. The user must accept the pull request in Github. There will be a test that runs to check if everything is correct. Once it the code is validated and confirmed as correct, the pull request will be completed. Simultaneously, the code will be deployed into a Heroku dummy app for testing. The user can then merge the pull request into the main branch if they wish. Once the request is merged, the user can rebase the code onto the main branch in their local coding software. Simultaneously, the code will be deployed into the final product application in Heroku.
