Arrange-Act-Assert

Summary

Arrange, Act, and Assert is considered the pattern to be followed when writing a testing code. Testing exercises a behavior to determine if the behavior functions correctly. It provides an empirical feedback loop for development. It allows programmers to know when their code breaks. This prevents program applications to be filled with bugs. Below is summary of each pattern:
-- Arrange: Arrange sets up the test case. It arranges all inputs and targets For example, this is where the objects are created or where a database is prepped.
-- Act: Act covers the main things to be tested. They act on the target behavior and elicit some sort of response. For example, this step could entail calling a function or method.
-- Assert: Assert verifies the goodness or badness of the response elicited by the act step. For example, assertions can check numeric or string values.
