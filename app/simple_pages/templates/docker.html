Docker Explanations

Containers

A container is an abstraction in the app layer which package code and some dependencies together. Containers take up little space in comparison to the virtual machine technologies. Furthermore, multiple containers can run on the same machine. They all run on the same OS kernel. Each container runs as an isolated process in the user space.

Image

A docker image is a file that is created in the coding environment. Once the coding is created, the docker image file is executed in a Docker container. They behave as the starting point when using Docker. The images have multiple layers and each one is built from the previous layer.

Linking a project repository on docker hub

One can link a project repository on docker hub by creating docker-compose.yml file and a regular docker file. The following code can be used to create a local container.
Docker-compose.yml file:
version: "3.2"
services:
flask:
build: .
restart: always
ports:
- 80:5000
stdin_open: true # docker run -i
tty: true # docker run -t
command: ["flask", "run"]
volumes:
- .:/home/myuser

Dockerfile
FROM python:3.8-buster
COPY requirements.txt .
ENV FLASK_APP=app
ENV PORT=5000
ENV FLASK_RUN_HOST=0.0.0.0
ENV FLASK_ENV=development
ENV PATH="/home/myuser/.local/bin:${PATH}"
RUN apt-get update &&\
/usr/local/bin/python3 -m pip install --upgrade pip &&\
/usr/local/bin/python3 -m pip install --upgrade setuptools &&\
/usr/local/bin/python3 -m pip install -r requirements.txt &&\
adduser myuser
WORKDIR /home/myuser
COPY --chown=myuser:myuser . .
CMD gunicorn -w 4 --bind 0.0.0.0:$PORT "app:create_app()"

Repositories

A Git Repository is known as the .git/folder for a project. The repository is what tracks all the changes made in a project. A coder must work on a branch, merge the branch to the main branch if they wish and then commit the work. Once work is committed, they can push the work into the repository. Many times, coders will commit and push a branch they are working on and check if the code in the repository is working correctly. Once they confirm the code is working in the repository, they can merge into the main branch if they wish.
