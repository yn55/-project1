Object Oriented Programming Glossary

Terms & Definitions

Object: In object-oriented programming, an object can be considered an instance of a particular class. Furthermore, an object consists of a state and a behavior. A state can be considered the adjective-type attribute of an object. For example, a dog has a name(State 1) and a color(State 2). A behavior can be considered the verb-type attribute of an object. For example, a dog can bark(Behavior 1) and can fetch(Behavior 2). You can refer to the following link for further information: https://docs.oracle.com/javase/tutorial/java/concepts/object.html

Instantiation:Creation of an object. You can refer to the following link for further information: https://www.techopedia.com/definition/26857/instantiate

Class: A class is a blueprint for creating objects. They can provide initial values for a state (considered member variables or attributes). They can provide implementations of behaviors (considered member functions or methods). A class is a way of organizing information about a type of data so a programmer can reuse elements when making multiple instances of that data type. It is an example of encapsulation which will be discussed in further detail later. You can refer to the following link for further information: https://brilliant.org/wiki/classes-oop/

Namespace: Creates a name for objects of various kinds. It allows for objects to have unique names for proper identification. You can refer to the following link for further information: https://en.cppreference.com/w/cpp/language/namespace

Constructor: A method of a class or structure that initializes the creation of an object. The constructor is called automatically when an object is created. It can be used to se the values of the members of an object. You can refer to the following link for further information: https://www.techopedia.com/definition/5656/constructor
