Git Explanations

Branch

A Git branch lists the user's project branches. The branch that is highlighted designates the branch that is currently in use. A user can type and develop code on a branch. This is a great way for users to make edits in a code program without damaging the finished project. It is generally advised for users not to work on the master branch (main branch). It is advised to work on a new branch and merge to the master once everything is confirmed as working.

Merge

Merge allows a user to integrate a branch with code into a another branch of code. Once the desired code is written on a branch, a user must click on the branch they wish to merge with and click merge. The current branch will be updated to reflect the merge. The target branch will remain unchanged. Only the merged branch will change.

Commit

Git Commit command takes all the code of the current project. The git command allows a user to save work at a current time frame. However, it does not make any actual changes to the project. It is a version of the project that can serve as a checkpoint. If a coder decides to add more code to the project but would like to revert to a past version, they can revert to the past version since they executed a git commit command previously.

Tags

Git Tags give coders the ability to have reference points in their Git History. They are used to create the versions of a code project. For example, if one version of a website is released to the public, they can use a git tag to mark it as "Version 1". If a coder would like to update the website and release an updated version of the website, they can create a new git tag and name it "Version 2".

Repositories

A Git Repository is known as the .git/folder for a project. The repository is what tracks all the changes made in a project. A coder must work on a branch, merge the branch to the main branch if they wish and then commit the work. Once work is committed, they can push the work into the repository. Many times, coders will commit and push a branch they are working on and check if the code in the repository is working correctly. Once they confirm the code is working in the repository, they can merge into the main branch if they wish.
